Short JavaScript notes and small code examples.
`let`, `var`, `const`

Cheat sheet
✍️ `var` is old and obsolete
✍️ `let`, `const` are modern
✍️ `let`: value can be changed
✍️ `const`: cannot be reassigned
✍️ `const` + primitive: nothing can be changed
✍️ `const` + object: you cannot assign a new object, but you can modify the existing one

Lecture
The keywords `var`, `let`, and `const` let you create a variable and assign an initial value. `var` is an old construct and now considered outdated. To create a variable with an immutable binding, use `const`. For a variable whose value will change, use `let`.

It is important to understand whether your value is a primitive or an object. `const a = 5` fully forbids any changes because `5` is a primitive type, and the statement "the value 5 is stored in variable `a`" remains true. But `const b = [5]` forbids writing `b = [6]`, while still allowing array element changes, for example `b[0] = 777`. This happens because the value of `b` is a reference to an array: we read that reference from the variable and then access the array by index.
