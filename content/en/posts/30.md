Short JavaScript notes and small code examples.
Hoisting

[the musketeers](<https://am-lang.web.app/assets/let_var_const_musketeers_hoisting-ChJi0L3X.webp>)

Cheat sheet

✍️ Hoisting moves declarations to the top of scope
✍️ `var` is accessible before declaration (`undefined`)
✍️ `let` and `const` are not accessible before declaration
✍️ The TDZ enforces this behavior

Lecture

In languages like C++, Java, and Go, you cannot use a variable before it is declared. JavaScript effectively scans the file in stages, processing declarations first.

Then the program runs, and declarations appear to be lifted upward. A [var](<https://github.com/AlinaBrode/js_snippets_blog/blob/main/hoisting/trap.js>) example may print `undefined`, while the "temporal dead zone" makes [let](<https://github.com/AlinaBrode/js_snippets_blog/blob/main/hoisting/let.js>) and [const](<https://github.com/AlinaBrode/js_snippets_blog/blob/main/hoisting/const.js>) throw an error.

The links point to a Visual Studio Code repository. To see how it works, just clone, open, and run it. If something does not work, let’s discuss it in the comments under the post.
